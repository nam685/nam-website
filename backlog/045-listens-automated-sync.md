# Automate daily listens sync (cron job)

The current manual sync flow is extremely painful:

1. Open browser, go to music.youtube.com
2. Open DevTools, click a song to trigger a POST request
3. Copy the POST request body (headers/cookies)
4. SSH into the Hetzner server
5. `cd` to deploy dir `nam-website-deploy`
6. Run `uv run ytmusicapi browser`, paste the POST body, Ctrl+D
7. Then trigger sync from the admin UI

This needs to be a daily cron job that runs automatically.

## Challenges
- ytmusicapi browser auth uses short-lived cookies that expire, requiring the manual copy-paste dance above
- The `browser.json` file generated by `ytmusicapi browser` goes stale and returns 403 after some time (see memory: "Custom Google OAuth gets 403 on get_history(); must use ytmusicapi's built-in OAuth")

## Possible approaches
1. **ytmusicapi OAuth flow** — ytmusicapi supports its own OAuth (`ytmusicapi oauth`). If the refresh token works long-term, this avoids the browser auth entirely. Need to test if `get_history()` works with OAuth tokens.
2. **Google OAuth with refresh token** — the site already has Google OAuth for watches. Could reuse a similar flow with the YouTube Music scope, store a refresh token, and auto-refresh.
3. **Headless browser automation** — last resort. Use Playwright/Selenium to log in and extract cookies on a schedule.

## Acceptance criteria
- Listens sync runs automatically once daily without manual intervention
- Auth tokens refresh automatically (no manual browser-paste needed)
- Failures are logged and optionally send a notification
- Add a systemd timer or Django management command triggered by cron
